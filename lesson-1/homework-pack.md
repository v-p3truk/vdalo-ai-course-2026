## A. Встановити агента

Окреме повідомлення в чаті — там швидкий гайд по 4 опціях (Codex / Claude Code / Antigravity / Cursor) + ціни + trial + спільні акаунти.

Якщо коротко:
- **Codex CLI** — `npm install -g @openai/codex`, потрібен ChatGPT Plus
- **Claude Code** — `npm install -g @anthropic-ai/claude-code`, Pro $20/міс або pay-per-use API
- **Antigravity** — https://antigravity.google.com, безкоштовно з робочою поштою
- **Cursor** — https://cursor.com, 14-денний Pro trial

Деталі і альтернативи — у [`tools-access-guide.md`](tools-access-guide.md).

→ Якщо щось не виходить з інсталом — спитай в команді, проведемо разом.

## B. Створити папку проєкту під клієнта

**Структура (мінімум):**

```
[client-name]/
├── AGENTS.md            ← головний файл контексту (заповнюємо в кроці C)
├── 01_strategy/         ← стратегія, дослідження
├── 03_client_materials/ ← фотки, відео, тексти від клієнта
├── 04_creatives/        ← креативи, банери
├── 05_reports/          ← звіти
└── 06_projects/         ← код, боти, лендінги (якщо є)
```

Можна створити через UI Finder/Explorer або в терміналі:
```bash
cd ~/Documents
mkdir -p MyClient/{01_strategy,03_client_materials,04_creatives,05_reports,06_projects}
```

> ⚠️ Файл контексту називається по-різному залежно від агента:
> - **Codex** → `AGENTS.md`
> - **Claude Code** → `CLAUDE.md`
> - **Antigravity** → `GEMINI.md`
> - **Cursor** → `.cursorrules`
>
> Зміст один і той самий — пишеш про продукт, клієнта, тон, KPI, як говоримо.

## C. Створити AGENTS.md з template

**Template (приклад) на GitHub:**
https://github.com/v-p3truk/vdalo-ai-course-2026/blob/main/templates/01_brand-context-template.md

→ Скопіюй файл у свою робочу папку → перейменуй у `AGENTS.md` (або `CLAUDE.md` / `GEMINI.md` / `.cursorrules`) → заповни.

**Що принципово важливо заповнити (а не "все підряд"):**

| Секція | Чому критично |
|--------|---------------|
| **01. Persona** | Без неї агент іде в generic AI-slop |
| **02. Бізнес-контекст** | Маржа, цикл, ринок — впливає на все |
| **03. ICP** | Біль клієнта реальними словами |
| **04. Voice DNA** | 3-5 рис + анти-приклади. Без анти — не працює |
| **05. Vocabulary** | "Завжди" / "ніколи" + як називаємо клієнта |
| **06. Few-shot examples** | 1-2 канонічні приклади > 10 правил |
| **08. Активні кампанії** | Якщо порожньо — агент пише "взагалі", не під поточний момент |

**Що можна пропустити на першому кроці:**
- Минулі кейси (10) — додати потім
- Заборонені теми (09) — додати коли побачиш проблему
- Структурні правила (07) — добре мати, але не блокери

**Принцип:** менше шуму = більше сигналу. **Не клади весь PDF brand book на 80 сторінок.** Тільки те що змінює рішення AI.

## D. Виконати 1 реальну задачу

Не тестову — реальну. Що завгодно з твоєї поточної роботи.

**Ідеї:**

| Задача | Що робить агент |
|--------|----------------|
| Креативний бриф під новий продукт | Читає AGENTS.md → генерує бриф у твоєму стилі |
| Швидкий репорт по кампанії | Завантажуєш CSV в папку → агент пише summary |
| Переробити ТЗ під новий формат | Старе ТЗ → агент адаптує під брендові правила |
| 5 заголовків для Meta Ad | Дає 5 варіантів з анти-прикладами що НЕ підходить |
| План публікацій на тиждень | Календар + ідеї постів |

Запускаєш агента у своїй папці → пишеш задачу природньою мовою. Все. Не треба писати "ти експерт..." — для цього і є `AGENTS.md`.

## E. Висновки — що було добре / що ні

Після виконання задачі — швидкий рев'ю результату:
- Що агент зрозумів правильно?
- Де він втратив контекст / промазав по тону?
- Що довелось правити руками?
- На якому етапі ти подумав "о, працює"?

Запиши швидко в `notes-l1.md` поруч.

## F. Оновити AGENTS.md ⭐

> Це найважливіший крок який часто пропускають.

Якщо агент зробив щось не так — це не його вина, це **відсутній або невірний контекст**. Виправляй це **в AGENTS.md**, не в наступному запиті.

**Що додати/виправити після першої задачі:**

| Якщо агент… | Додай в AGENTS.md |
|------------|-------------------|
| Промазав по тону | Конкретний приклад в 04. Voice DNA: "✅ on-brand: ..." / "❌ off-brand: ..." |
| Неправильно назвав клієнта/продукт | 05. Vocabulary — "як називаємо: ..." |
| Дав загальні поради замість конкретики | 02. Бізнес-контекст — додай нюанс ринку |
| Не знав про активний офер | 08. Активні кампанії — додай рядок |
| Згадав щось чого нема | 09. Заборонені теми — додай |

**Це і є секрет роботи з агентом:** AGENTS.md — це *жива* база яка росте з кожною задачею. Друга задача буде вже легшою. Десята — як магія.

```markdown
1. ...
2. ...

## ❌ Де агент промазав
1. ...
2. ...

## 🔄 Що оновив у AGENTS.md після задачі
- ...
- ...

## ❓ Питання що залишились
- ...

## 🤔 Що зрозуміло / що ні з Уроку 1
**Зрозуміло:**
- ...
**Не зрозуміло:**
- ...
```

→ Принеси на Урок 2 (15.05) — розберемо разом, відповім на не зрозуміле.

## FAQ

**В: А якщо я ніколи не працював з агентом / терміналом?**
Перші 30 хв виглядатиме незвично — потім норм. Antigravity і Cursor — це IDE (графічний редактор), там простіше старт. Codex/Claude Code — термінал, але теж нестрашно. Якщо стопор — спитай в команді.

**В: А якщо мій клієнт не маркетинговий?**
Структура папки і AGENTS.md універсальні — заповнюєш під свій випадок.

**В: Чи треба знати програмування?**
**Ні.** Ми використовуємо AI на природній мові.

**В: Скільки коштує?**
Codex / Claude Code / Cursor — $20/міс. Antigravity — безкоштовно. Якщо ваша агенція має корпоративну підписку — уточнюйте у адміна.

**В: А якщо я плутаюсь — куди звертатись?**
В команду курсу або issues цього repo. "Дурних" питань на цьому етапі не існує.
